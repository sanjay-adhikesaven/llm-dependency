# Describe Entities

Descriptions are generated for concrete entity leaves only. Use exact
anchor metadata, source evidence, and reviewed aux fields. Do not
describe broad concept/projection nodes as if they were releases.

Description rules:

- Prefer HF README front matter and model/dataset card prose for HF
  anchors.
- Use GitHub release paths, API model ids, official release pages, or
  exact paper-only records when HF is not available.
- Include numeric/source-stated facts when present.
- Keep descriptions neutral; do not frame an upstream artifact only as
  data or a base model for the current target.
